# Code Standards

## Principles

| Principle | Meaning |
|-----------|---------|
| **YAGNI** | Don't build features until needed |
| **KISS** | Prefer simple solutions |
| **DRY** | Extract shared logic |

## General Rules

| Rule | Standard |
|------|----------|
| File names | `kebab-case` |
| File size | < 200 lines |
| Git commits | `type(scope): description` |

**Commit types:** `feat`, `fix`, `refactor`, `docs`, `style`, `test`, `chore`

## Pre-Commit Checklist

- [ ] Build passes
- [ ] No TypeScript errors
- [ ] No `console.log` statements
- [ ] No secrets committed
